# 프로젝트 요구사항 정의서 (PRD)

본 문서는 프리미엄 유틸리티 웹사이트(로또 생성기, 동물상 테스트, 블로그)의 필수 기능 명세와 **구글 애드센스 고품질 가이드라인**을 준수하기 위한 기획/개발 지침입니다. 개발 시 본 문서의 내용을 반드시 반영해야 합니다.

## 1. 사이트 목적 및 디자인 컨셉
- **목적**: 방문자에게 유용한 도구(로또, 동물상)와 정보(블로그)를 제공하여 체류 시간을 늘리고, 구글 애드센스 수익을 창출하는 고품질 웹사이트.
- **디자인 컨셉**: 최신 B2B SaaS 느낌의 '라이트/퍼플 핀테크(Light/Purple Fintech)' 테마를 유지. 부드러운 화이트 글래스모피즘 뷰와 3D 오브젝트(Spline)를 결합하여 사용자에게 높은 신뢰감을 제공.

## 2. 필수 구현 기능 (Core Features)

### 2.1 로또 번호 추천기 (Lotto Generator)
- **기능**: 사용자가 '생성하기' 버튼을 클릭하면 1부터 45까지의 숫자 중 중복 없는 6개 번호를 5세트 생성하여 화면에 표시.
- **UX**: 숫자가 생성될 때 부드러운 애니메이션 효과를 주어 시각적 즐거움 제공.

### 2.2 로또 당첨 이력 및 통계 조회 (History & Stats)
- **기능 1 (당첨 이력)**: 실제 과거 로또 당첨 번호(모의 데이터 또는 API 활용)의 회차별 당첨 번호 목록을 테이블 또는 리스트 형태로 제공.
- **기능 2 (통계 그래프)**: 최근 N회차 동안 가장 많이 나온 번호의 빈도수 및 점유율을 **그래프(Chart.js 등 활용)**로 시각화하여 정보성 콘텐츠로 제공.

### 2.3 동물상 테스트 (Animal Face Test)
- **기능**: 사용자가 자신의 사진을 드래그 앤 드롭(Drag & Drop)하여 업로드하면, 사진을 분석하여 어떤 동물상(강아지, 고양이, 여우 등)인지 결과를 알려주는 기능.
- **구현**: 파일 업로드 UI 컴포넌트(점선 테두리 박스 등) 구현. 실제 분석은 Teachable Machine 모델이나 모의(Mock) 로직을 연동.

### 2.4 블로그 (Blog System)
- **기능**: 정보성 글(아티클)을 리스트 형태로 보여주는 페이지.
- **구성**: 각 아티클은 썸네일, 제목, 요약 내용, 작성일자를 포함하는 깔끔한 카드 형태로 리스트업 됨. SEO 및 애드센스 승인에 유리한 텍스트 위주의 정보 제공.

## 3. 구글 애드센스 고품질 가이드라인 준수 지침

구글 애드센스 승인 및 유지 보수를 위해 다음의 원칙을 모든 페이지에 적용합니다.

1. **독창적이고 풍부한 텍스트 콘텐츠 제공**:
   - 도구(로또, 동물상)만 달랑 두지 않고, 해당 페이지 하단이나 주변에 "로또 번호 생성 원리", "통계 분석 방법", "동물상 테스트란?" 등 상세한 텍스트 설명을 추가하여 '정보성(Information)'을 띠게 합니다.
2. **명확한 네비게이션 및 사이트 구조 (UX)**:
   - 사용자가 원하는 도구를 쉽게 찾을 수 있도록 직관적인 GNB(상단 메뉴)와 상세한 Footer 구조를 유지합니다.
3. **페이지 로딩 속도 최적화 (Web Vitals)**:
   - 3D 오브젝트나 이미지가 페이지 렌더링을 심각하게 방해하지 않도록 지연 로딩(Lazy loading) 등을 고려하고, CSS/JS 코드를 깔끔하게 유지합니다.
4. **신뢰성 페이지 필수 포함**:
   - Footer에 개인정보처리방침(Privacy Policy), 서비스 이용약관, 면책 조항(Disclaimer - 예: "로또 생성 결과는 당첨을 보장하지 않습니다.")을 명확히 표기합니다.
5. **모바일 최적화 (Responsive Design)**:
   - 스마트폰 화면에서도 레이아웃이 깨지지 않고 완벽하게 동작하도록 반응형(Media Queries) 설계를 적용합니다.
